Para integrar la facturación de Alegra en Argentina, el portal centralizado para desarrolladores es el mismo para todos los países, pero con especificaciones críticas para cumplir con la AFIP.
Aquí tienes los enlaces y puntos clave para tu ERP:
1. Enlaces Principales
• Documentación Oficial: Alegra API Documentation
• Referencia de Facturas (POST): Crear Factura de Venta
• Catálogo Específico de Argentina: Parámetros Argentina (Aquí encontrarás los códigos de condición de IVA, tipos de comprobantes y documentos de identidad válidos para AFIP).

2. Autenticación
Alegra utiliza Basic Auth. Necesitarás:
1. El correo electrónico de la cuenta.
2. El API Token (lo generas en Configuración -> API dentro de tu cuenta de Alegra).
Bash
# Ejemplo de Header
Authorization: Basic base64(email:api_token)

3. Consideraciones Críticas para Argentina (AFIP)
Para que una factura sea válida ante la AFIP desde la API, debes prestar atención a estos campos en el JSON que envíes:
• stamp: Debe estar en true si quieres que la factura se emita y autorice automáticamente ante la AFIP (obteniendo el CAE).
• numberTemplate: Debes enviar el id de la numeración que configuraste como "Factura Electrónica" en Alegra.
• Datos del Cliente: El campo idIdentification debe ser CUIT, CUIL o DNI y el identification debe ser el número sin guiones.
• Condición de IVA: Es obligatorio enviar la condición tanto del emisor como del receptor (ej: Responsable Inscripto, Monotributista).
4. Flujo de Implementación Recomendado
1. Obtener Numeraciones: Llama a GET /api/v1/number-templates para identificar cuál es la que corresponde a Factura A, B o C electrónica.
2. Sincronizar Ítems: Usa GET /api/v1/items para obtener los IDs de tus servicios/productos.
3. Emitir Factura: Realiza el POST /api/v1/invoices con el flag stamp: true.

Esta es una excelente estrategia para mantener Supabase como tu "única fuente de verdad". Para lograr que Alegra emita la factura electrónica (CAE de AFIP) usando tus datos de Supabase y luego guarde el PDF resultante en tu Storage, el flujo técnico es el siguiente:
1. El Concepto: Ítems "On-the-fly"
A diferencia de otros países, en la API de Alegra para Argentina, no necesitas crear el producto previamente en Alegra. Puedes enviar los datos de tus servicios (peluquería) o productos (perfumería) directamente en el array de items al crear la factura.
2. Estructura de la Petición (Edge Function)
Deberás crear una Edge Function que tome los datos de tu tabla appointments o sales y haga un POST a Alegra.
Campos obligatorios para AFIP:
• stamp: true: Es lo que le dice a Alegra: "conéctate con AFIP ahora mismo y pide el CAE".
• numberTemplate: El ID del punto de venta electrónico configurado en Alegra.
• client: Debes enviar el CUIT/CUIL del cliente. Si es consumidor final, Alegra ya tiene un ID por defecto para eso.
TypeScript
// Ejemplo del JSON que enviaría tu Edge Function a Alegra
{
  "date": "2026-03-01",
  "dueDate": "2026-03-01",
  "client": { "id": "ID_DEL_CLIENTE_EN_ALEGRA" },
  "items": [
    {
      "name": "Corte de Cabello + Estilismo", // Traído de Supabase
      "price": 15000,
      "quantity": 1,
      "tax": [{ "id": "1" }] // ID del IVA 21% en Alegra Argentina
    }
  ],
  "stamp": true, // <--- CRÍTICO para Factura Electrónica
  "type": "sale",
  "numberTemplate": { "id": "1" } // ID de tu punto de venta AFIP
}

3. Obtener el PDF y guardarlo en Supabase
Una vez que el POST es exitoso, Alegra te devolverá el id de la factura y el objeto stamp con los datos de AFIP.
Para llevar el PDF a tu Storage:
1. Obtener URL del PDF: Llama al endpoint GET /api/v1/invoices/{id}. En la respuesta, busca el campo pdfUrl (o usa fields=url).
2. Descargar y Subir: Tu Edge Function debe hacer un fetch a esa URL para obtener el archivo como un blob y luego usar el cliente de Supabase para subirlo:
TypeScript
const pdfBlob = await fetch(alegraResponse.url).then(r => r.blob());

await supabase.storage
  .from('invoices')
  .upload(`${invoiceId}.pdf`, pdfBlob);
4. Configuración Previa en Alegra
Antes de programar, debes hacer esto manualmente en el portal de Alegra:
1. Habilitar API: En Configuración -> API, obtén tu correo y API Token.
2. Configurar AFIP: Asegúrate de que Alegra ya esté vinculado con tus certificados de AFIP y que tengas un "Número de comprobante" (Punto de Venta) activo.
3. Identificar Impuestos: Haz un GET /api/v1/taxes para saber cuál es el ID del IVA 21%, 10.5%, etc., en tu cuenta.

se realicen deben incluir el header Authorization.
El acceso al API se realiza utilizando el correo y el token otorgado al usuario registrado en Alegra.
Obtener token de acceso
Debes seguir los siguientes pasos para obtener el token de acceso:
	1. Ingresar a aplicación Alegra.
	2. Haz clic sobre el vínculo "Configuración" en la parte superior derecha de la pantalla de Alegra y haz clic en la sección "API - Integraciones con otros sistemas"
	3. En la nueva pantalla puedes encontrar el correo con el cual debes acceder al API y el token. Si aún no cuentas con un token puedes generarlo también.
¿Cómo utilizar el token?
En el header Authorization se debe poner el correo y el token del usuario separado por dos puntos (:), todo en base64.
Por ejemplo, si el correo del usuario es ejemploapi@alegra.com y el token es tokenejemploapi12345 el header Authorization debe quedar así :
Authorization: Basic ZWplbXBsb2FwaUBhbGVncmEuY29tOnRva2VuZWplbXBsb2FwaTEyMzQ1
Donde ZWplbXBsb2FwaUBhbGVncmEuY29tOnRva2VuZWplbXBsb2FwaTEyMzQ1 es base_64('ejemploapi@alegra.com:tokenejemploapi12345')
Errores en autenticación
Si el API encuentra algún inconveniente con la autenticación retorna un código 401 y el error encontrado.
